# セキュリティについて

[繁體中文](SECURITY.md)｜[English](SECURITY.en.md)｜日本語

## セキュリティ境界

- サービスは初期状態で停止しており、利用者が眼鏡のフォアグラウンド画面から操作した場合に限り起動します。
- USB モードはデバイスの loopback だけを待ち受け、Wi‑Fi モードは現在のプライベート IPv4 だけを待ち受けます。
- 起動するたびに、ペアリングコードと高エントロピーのセッショントークンを再生成します。認証失敗の試行はレート制限され、10 分間操作がなければ自動停止します。
- Web API はサーバー側の opaque ID を使用し、クライアントからファイルパスを受け付けません。直接ファイルモードでは canonical path とシンボリックリンクを検証し、MediaStore の代替動作ではプライマリボリューム、相対パスのホワイトリスト、実際の読み取り可否を検証します。
- 変更操作には、同一オリジンのセッション Cookie とカスタム CSRF header が必要です。レスポンスには CSP、`frame-ancestors 'none'`、`nosniff`、`no-store` を使用します。
- アップロードは一時ファイルへ完全に書き込み、`fsync` を実行した後で正式名へ移動します。同名の既存ファイルは上書きしません。削除操作の初期動作はゴミ箱への移動です。
- アプリは、カメラ、マイク、位置情報、連絡先、広告 ID、自動起動の権限を要求しません。また、外部ホストへ接続しません。
- QR コードは外部サービスを使わず、眼鏡内だけで生成します。画面の wake lock は共有中だけ保持し、共有停止またはタイムアウト時に解放します。

## HTTP の制限

ブラウザがローカルネットワーク上の IP へ直接接続する場合、「完全なオフライン動作」「証明書の警告なし」「公開 CA による検証」の 3 条件を同時に満たすことはできません。本バージョンの Wi‑Fi モードは HTTP を使用するため、PIN／Cookie は操作権限の認証だけを行い、**通信の機密性を提供せず、同じネットワーク上の中間者攻撃を防止できません**。

推奨する優先順位：

1. 機密性の高いファイルには USB＋ADB forwarding を使用します。
2. Wi‑Fi は自分のスマートフォンのテザリングだけで使用し、他のクライアントが接続していないことを確認します。
3. 公共または企業の共有ネットワークでは、共有を開始しません。
4. 作業終了後、直ちにサービスを停止します。

将来、「データを保存しないスマートフォン用操作アプリ」を別途開発する場合、Android Keystore でデバイス証明書を生成し、スマートフォン側で certificate pinning を行うことで、警告のない操作性とエンドツーエンド暗号化を両立できます。

## 保護の対象外

- root 化済み、悪意のあるシステムアプリに制御されている、またはファームウェアが置き換えられた眼鏡。
- 利用者が信頼済みとして許可した ADB ホスト。
- 利用者が自らダウンロードした後の、スマートフォン／パソコン上のコピーとブラウザキャッシュ。
- Rokid 純正カメラまたは Hi Rokid 自体によるデータ処理。

## 問題の報告

公開 Issue に、実際の写真、動画、ペアリングコード、IP、シリアル番号、完全な log を投稿しないでください。報告には、アプリバージョン、YodaOS バージョン、Android API、再現手順、匿名化済みのエラーだけを含めてください。
