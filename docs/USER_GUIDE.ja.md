# 眼鏡ファイルステーション 使用ガイド

[繁體中文](USER_GUIDE.md)｜[English](USER_GUIDE.en.md)｜日本語

## 最初に知っておくべき 3 つのこと

1. **スマートフォンへファイル管理アプリを別途インストールする必要はありません。** Safari、Chrome、その他のモダンブラウザを使用できます。
2. **アルバムを事前同期しません。** 一覧とサムネイルは必要なときだけ読み込まれます。ダウンロードを押した場合に限り、スマートフォンまたはパソコン上へコピーが作成されます。
3. **機密性の高いファイルには USB を優先してください。** Wi‑Fi モードは、自分のスマートフォンのテザリングまたは信頼できるネットワークだけで使用してください。

## 画面言語と実際の表示名

現在の APK とブラウザ画面は繁体字中国語です。本ガイドの日本語は意味を説明するためのもので、実際の画面では次の表示名を使用してください。

| 意味 | アプリに表示される名称 |
|---|---|
| アプリのランチャー名 | `眼鏡檔案站` |
| ファイル管理を許可 | `允許檔案管理` |
| USB パソコン管理 | `USB 電腦管理` |
| Wi‑Fi スマートフォン管理 | `Wi‑Fi 手機管理` |
| 共有を停止 | `停止分享` |
| 保護に設定／保護を解除 | `設為保護` / `解除保護` |
| ゴミ箱へ移動／復元 | `移到垃圾桶` / `還原` |

## 初回インストール

### 方法 A：Hi Rokid Toolbox

1. Hi Rokid と眼鏡の YodaOS-Sprite を更新します。
2. Hi Rokid で Toolbox を開きます。
3. 提供された `GlassesFiles.apk` を選択し、眼鏡へインストールします。
4. 眼鏡を装着し、アプリ一覧の末尾から `眼鏡檔案站` を探します。

Hi Rokid の画面は、提供地域によって多少異なる場合があります。Toolbox がない場合は、方法 B を使用してください。

### 方法 B：ADB デバッグケーブル

1. データ端子を備えた Rokid デバッグケーブルを用意します。同梱の充電ケーブルは、データ通信に対応していない場合があります。
2. Hi Rokid／眼鏡の設定で、開発者モードと ADB debugging を有効にします。
3. パソコンへ接続し、次のコマンドで確認します。

```bash
adb devices -l
```

4. APK をインストールします。

```bash
adb install -r GlassesFiles.apk
```

## 初回の権限設定

1. `眼鏡檔案站` を開きます。
2. `允許檔案管理`（ファイル管理を許可）を選択します。
3. Android の設定画面で「すべてのファイルへのアクセス」を許可します。
4. アプリへ戻ります。この権限は保持されるため、毎回設定し直す必要はありません。

アプリはファイル管理権限を取得しますが、実際に管理する対象は `DCIM/Camera`、`DCIM/album`、`Pictures`、`Movies` などのメディア用ホワイトリストだけです。システムファイルや他のアプリの非公開データを閲覧しません。

## 眼鏡上での操作

| 操作 | 機能 |
|---|---|
| 前方／後方へスワイプ | 前の項目／次の項目 |
| タップ | 現在の項目を開く、またはボタンを実行 |
| ダブルタップ | 1 つ前の画面へ戻る |
| 長押し | 項目メニューを開く（ファームウェア側で AI 用に予約されている場合は、タップ後の操作画面を使用） |
| 物理的な戻るボタン／DPAD | Android 標準の戻る／方向キーと同じ |

ホーム画面の機能：

- 今日撮影したファイル
- すべての写真と動画
- 写真のみ／動画のみ
- 保護済み
- お気に入り
- 大容量ファイル
- 重複ファイル
- ゴミ箱
- USB パソコン管理
- Wi‑Fi スマートフォン管理
- ストレージ権限と空き容量の情報

### 重要なファイルを保護する

ファイル操作画面で「保護に設定」を選択します。保護された項目は、名前変更やゴミ箱への移動ができません。操作するには、先に手動で保護を解除する必要があります。この設定は眼鏡上のアプリ内設定だけに保存され、元のメディア内容は変更しません。

### 削除と復元

- 通常の「削除」は、復元可能なゴミ箱へ移動するだけです。
- ゴミ箱から元の場所へ復元できます。同じ名前がすでに使われている場合、アプリは処理を中止し、先に競合を解消するよう案内します。

## スマートフォンのブラウザによる管理と遠隔操作

1. 自分のスマートフォンでテザリングを開始します。
2. 眼鏡をそのテザリングへ接続します。
3. 眼鏡のホーム画面で「Wi‑Fi スマートフォン管理」を選択します。
4. スマートフォンのカメラで眼鏡に表示されたオフライン QR コードを読み取ります。認識できない場合は、`http://192.168.43.20:8765` などの URL を手入力します。
5. その都度生成された 6 桁のペアリングコードを入力します。

共有とスマートフォン操作が有効な間は、リモート操作の結果を確認できるよう眼鏡画面が常時点灯します。共有停止または 10 分間の無操作タイムアウト後は通常のスリープ動作へ戻ります。QR コードは眼鏡内で生成され、URL が外部サービスへ送信されることはありません。

ペアリング後にできること：

- 今日、指定日、写真、動画、保護済み、お気に入り、大容量、重複グループで絞り込み。
- 直接ストリーミングしてプレビュー。動画は Range に対応し、シーク操作や切断後に動画全体を再転送する必要はありません。
- 項目ごとに保護／お気に入りの切り替え、名前変更、ゴミ箱への移動、ダウンロード。
- 「眼鏡リモコン」にある「前へ」「次へ」「開く」「戻る」の 4 つの大きなボタンで、眼鏡画面を操作。

終了後、眼鏡で「共有を停止」を選択してください。10 分間操作がない場合も、サービスは自動停止します。

## USB でパソコンから管理

1. 眼鏡のホーム画面で「USB パソコン管理」を選択します。
2. デバッグケーブルを接続し、次を実行します。

```bash
adb forward --no-rebind tcp:8765 tcp:8765
```

3. パソコンのブラウザで `http://127.0.0.1:8765` を開きます。
4. 眼鏡に表示されたペアリングコードを入力します。
5. 終了後、共有を停止して port forwarding を解除します。

```bash
adb forward --remove tcp:8765
```

## 重複ファイルと大容量ファイルの整理

- 大容量ファイルは、ファイルサイズで直接並べ替えるため、先に内容全体を読み込む必要はありません。
- 重複ファイルのスキャンでは、まずサイズ別にグループ化した後、小容量の固定バッファを使ってストリーミングしながら SHA‑256 を計算します。動画全体を RAM へ読み込みません。
- 初期設定では、充電中だけスキャンします。充電していない場合も手動で開始できますが、眼鏡のバッテリー消費や発熱が増える可能性があります。
- スキャンはキャンセルできます。キャンセルしても元のファイルは変更されません。
- 同じハッシュ値のグループでも、残すファイルは利用者が判断します。アプリが自動削除することはありません。

## よくある質問

### スマートフォンから URL へ接続できない

- スマートフォンと眼鏡が同じ個人用テザリング／Wi‑Fi に接続されていることを確認します。
- 以前のブックマークではなく、その時点で眼鏡に表示されている IP を使用します。
- スマートフォンの VPN やローカルネットワークを遮断するセキュリティツールを停止してから、もう一度試します。
- 共有を一度停止してから再開します。ペアリングコードも同時に更新されます。

### ファイルは見えるが、名前変更や削除ができない

現在、MediaStore の読み取り専用代替動作が使用されています。ホーム画面へ戻って「ファイル管理を許可」を選択し、Android の設定で「すべてのファイルへのアクセス」が有効になっていることを確認してください。

### 撮影したばかりの写真が見つからない

ホーム画面へ戻って更新してください。それでも見つからない場合は、[実機検証チェックリスト](DEVICE_CHECK.ja.md)に従って、純正カメラが実際に使用するパスと MediaStore データを確認してください。

### ブラウザで動画を再生できない

多くの場合、ブラウザが動画のコーデックに対応していないことが原因であり、ダウンロードファイルの破損を意味するものではありません。先に原本をダウンロードし、そのコーデックに対応するプレーヤーで開いてください。

### アプリがシステムによって停止される

眼鏡のバッテリー残量低下、折りたたみによるスリープ、YodaOS の省電力機能によって、フォアグラウンドサービスが停止する場合があります。眼鏡を再び装着し、手動で共有を再開してください。常駐するための自動起動設定は行いません。
