# 安全性說明

繁體中文｜[English](SECURITY.en.md)｜[日本語](SECURITY.ja.md)

## 安全邊界

- 服務預設關閉，只能由使用者在眼鏡前景畫面啟動。
- USB 模式只監聽裝置 loopback；Wi‑Fi 模式只監聽當下的私有 IPv4。
- 每次啟動重新產生配對碼及高熵工作階段權杖；失敗嘗試會限速，閒置 10 分鐘自動停止。
- 網頁 API 使用伺服器端 opaque ID，不接受客戶端檔案路徑；直接檔案模式執行 canonical path／符號連結驗證，MediaStore 備援則驗證主要儲存區、相對路徑白名單與實際可讀性。
- 修改操作要求同源工作階段 Cookie 與自訂 CSRF header；回應使用 CSP、`frame-ancestors 'none'`、`nosniff` 與 `no-store`。
- 上傳先完整寫入暫存檔並執行 `fsync`，之後才移入正式檔名，且不覆寫既有同名檔；刪除預設移到垃圾桶。
- App 不要求相機、麥克風、定位、聯絡人、廣告 ID 或開機啟動權限；不連線到任何外部主機。
- 螢幕喚醒鎖只在分享期間持有，停止或閒置逾時後釋放。

## HTTP 限制

瀏覽器直接連區網 IP 時，無法同時做到「完全離線、無憑證警告、由公開 CA 驗證」三件事。本版本 Wi‑Fi 模式採 HTTP，因此 PIN／Cookie 只負責驗證操作權限，**不提供傳輸機密性或對抗同網段中間人攻擊**。

建議順序：

1. 敏感檔案使用 USB＋ADB forwarding。
2. Wi‑Fi 僅使用自己的手機熱點，確認沒有其他用戶端。
3. 不在公共或企業共用網路啟動分享。
4. 完成後立即停止服務。

未來若另做「不保存資料的手機控制 App」，可用 Android Keystore 產生裝置憑證並在手機端做 certificate pinning，兼顧無警告體驗與端對端加密。

## 不在防護範圍

- 已 root、遭惡意系統 App 控制或被替換韌體的眼鏡。
- 已獲使用者信任的 ADB 主機。
- 使用者主動下載後，手機／電腦上的副本與瀏覽器快取。
- Rokid 原廠相機或 Hi Rokid 自身的資料處理行為。
